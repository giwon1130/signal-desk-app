## 문서 작성 원칙

1. **첫 줄에 "누가 봄 / 언제 봄"** 명시
2. **모든 결정에 "왜"** — 코드는 "무엇을" 만 알려줌. 문서는 "왜" 의 SoT
3. **링크 우선, 복붙 금지** — 정의는 한 곳 (예: 용어는 `용어집.md` 만)
4. **반말체 + 한국어** — 앱 본문 톤과 일치
