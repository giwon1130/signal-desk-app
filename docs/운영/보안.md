## 인증 흐름

(stub)
### 이메일 / 비밀번호
1. 클라 → `POST /auth/signup` (또는 `/auth/login`)
2. 서버: BCrypt 해시 비교
3. JWT 발급 (HS256, 720h)
4. 클라: AsyncStorage / localStorage 저장

### Google OAuth
1. 클라: GIS (web) 또는 native SDK (iOS) → `idToken`
2. 클라 → `POST /auth/oauth/google` body `{ idToken }`
3. 서버: `tokeninfo` 호출 → `aud` 화이트리스트 검증 → `sub` 로 사용자 매칭/생성
4. 자체 JWT 발급

## JWT

(stub)
- 알고리즘: HS256
- secret: `SIGNAL_DESK_JWT_SECRET` (운영 32자+ 랜덤)
- 만료: 720h (30일)
- claims: `sub` (UUID) + `email`
- **rotation**: secret 교체 시 모든 사용자 토큰 무효화 — 재로그인 필요. 사용자 영향 알리고 새벽에.
