## 수동 배포 (트러블슈팅 / 강제)

```bash
cd signal-desk-api
railway link --project signal-desk-and-otaku-feed --environment production --service signal-desk-api
railway up --service signal-desk-api --ci --detach

# 같은 식으로 web
cd signal-desk-app
railway up --service signal-desk-web --ci --detach
```

## 새 환경변수 추가 절차

(stub)
1. [환경변수.md](환경변수.md) 에 변수 추가 (이름 + 의미 + 예시)
2. Railway 대시보드 또는 `railway variables --set KEY=VALUE`
3. 코드에서 `@Value("\${...}")` (BE) 또는 `EXPO_PUBLIC_*` (FE) 로 참조
4. 부팅 시 검증 로그 추가 (예: secret 더미값 경고)

## 검증 (배포 후 30초)

```bash
# API
curl -s -o /dev/null -w "%{http_code}\n" https://signal-desk-api-production.up.railway.app/health

# Web (번들 해시 변경 확인)
curl -s https://signal-desk-web-production.up.railway.app/ | grep -oE 'index-[a-f0-9]+\.js' | head -1

# CORS
curl -s -o /dev/null -w "%{http_code}\n" -X OPTIONS \
  -H "Origin: https://signal-desk-web-production.up.railway.app" \
  -H "Access-Control-Request-Method: GET" \
  https://signal-desk-api-production.up.railway.app/api/v1/market/summary
```

## 자주 발생하는 문제

(stub)
| 증상 | 원인 | 해결 |
|------|------|------|
| 빌드는 됐는데 옛 버전 보임 | startCommand 가 Dockerfile CMD 덮어씀 | railway.toml 의 startCommand 제거 |
| /auth/* 모두 404 | file 모드 (JDBC 비활성) | `SIGNAL_DESK_STORE_MODE=jdbc` 설정 |
| /api/* 403 | CORS 외부 도메인 | `SIGNAL_DESK_CORS_ALLOWED_ORIGIN_PATTERNS` 추가 |
| /auth 로그인 429 | rate limit | 1분 대기 또는 다른 IP |
| 구글 로그인 invalid_client | GCP origin 미등록 | GCP OAuth Client 의 Authorized JS origins 에 도메인 추가 |
