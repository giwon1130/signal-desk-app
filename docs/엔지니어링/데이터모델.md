# 데이터 모델

> **누가 봄**: FE + BE
> **언제 봄**: 새 필드 추가 / 응답 shape 바꿀 때
> **규칙**: BE Kotlin 타입 (`MarketOverviewModel.kt`) ↔ FE TS 타입 (`src/types/`) **1:1 매칭**.
> 한쪽만 바꾸면 안 됨 — 두 저장소 동시 PR.
> FE 타입은 도메인별 파일로 분리: `types/market`·`types/ai`·`types/workspace`·`types/briefing`·`types/media`·`types/system`. `src/types.ts` 는 barrel(re-export) 이라 `from '../types'` 도 호환.

## 주요 필드 의미

(stub)
- `expectedReturnRate` — **percent 그대로** (예: 4.6 = 4.6%)
- `realizedReturnRate` — **percent 그대로**
- `changeRate` — **percent 그대로**
- `confidence` — 0~100 (정수)
- `entryPrice / stopLoss / takeProfit` — 정수 (KR 원, US 센트)
- `publishedAt` — ISO-8601 UTC 문자열
