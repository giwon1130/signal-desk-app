## 단계 0 — TestFlight Processing 대기

`eas submit` 끝난 후 App Store Connect 의 **TestFlight** 탭에 빌드 진입.
- "Processing" 표시 → 15분~1시간 후 "Ready to Test" 로 전환
- 그 전엔 메타데이터 attach 못 함

확인: https://appstoreconnect.apple.com/apps/6767399371/testflight/ios

## 단계 1 — 본인 디바이스 베타 검증 (필수)

**TestFlight** 앱으로 본인 폰에 설치해서 한 번 돌려야 함.

1. iPhone 에 **TestFlight** 앱 설치 (App Store)
2. App Store Connect → **TestFlight** → **Internal Testing** → 본인 추가
3. iPhone TestFlight 앱에서 받은 초대 수락
4. 빌드 다운로드 → 동작 확인

**최소 검증 항목**:
- [ ] 회원가입 / 로그인 (이메일 + Google)
- [ ] 알림 권한 prompt → 허용
- [ ] 시장 데이터 로드
- [ ] 관심종목 추가/해제
- [ ] 종목 검색
- [ ] 다크모드 전환

문제 있으면 → 코드 고치고 재빌드 (`eas build` 재실행, version code 자동 증가).

## 단계 6 — Submit for Review

전체 메타데이터 + 스크린샷 + Build + Privacy 검수 끝나면:

오른쪽 상단 **"Add for Review"** → **"Submit to App Review"**

### 심사 기간
- **평균 24~48시간** (영업일 기준)
- 빠르면 12시간, 늦으면 1주
- 거절(reject) 되면 사유 + 수정 요청 → 재제출

---

## 거절 사유 자주 나오는 것

### Guideline 5.1.1 — Privacy Policy
- Privacy Policy URL 페이지 접근 안 됨 → 검증: `curl https://.../privacy` → 200
- Privacy Policy 와 App Privacy 양식 불일치

### Guideline 2.1 — App Crashed
- TestFlight 베타로 본인 검증 필수

### Guideline 4.0 — Design / Spam
- "투자 가이드" 라고 강조하면서 실제 투자 자문 자격 없으면 안 됨
- 우리 앱은 "정보 제공" 임을 명확히 (description 에 disclaimer)

### Guideline 5.1.2 — Account Sign-In
- Demo account 동작 안 함
- 또는 Google 로그인이 reviewer 환경에서 안 됨

### Demo Account 막힘
- Apple reviewer 가 IP 다양해서 우리 rate limit 걸릴 수 있음
- Submit 직전엔 rate limit 잠시 풀어주거나 reviewer IP 화이트리스트 (별도 endpoint)

## 거절 받으면

1. App Store Connect → Resolution Center → 거절 사유 읽기
2. 코드 / 메타데이터 수정
3. 새 빌드 (`eas build` — 자동 versionCode 증가)
4. 또는 메타데이터만 수정 후 다시 Submit
