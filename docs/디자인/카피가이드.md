# 카피 가이드

> **누가 봄**: 디자이너 / FE / PM
> **언제 봄**: 새 텍스트 쓰기 전

## 톤

(stub)
- **반말체** (~해, ~봐, ~줘) — 친근. 존댓말 금지 (운영 안내 정도만 예외).
- **한국어 우선** — 불가피한 영어는 통일 (예: AI, NASDAQ, OPEN/CLOSED).
- **짧게** — 한 줄에 한 메시지.

## 자주 쓰는 표현 표준화

(stub)
| 상황 | 권장 | 금지 |
|------|------|------|
| 빈 관심종목 | "아직 관심종목이 없어" | "관심종목이 비어있습니다" |
| 보유 등록 안내 | "종목 상세에서 매수가 · 수량 입력해 등록" | |
| AI 추천 준비 중 | "AI 추천 준비 중" + "오늘 픽이 산출되면 여기 떠" | |
| 휴장 모드 | "지금 휴장이라 ..." | "장이 닫혀있어" |
| 손익 가이드 | "익절 구간 — 분할 매도 고려" | "팔아라" |

## 금지 표현

(stub)
- "추천드립니다" / "확인부탁드립니다" — 존댓말
- "꼭", "반드시", "절대" — 단정/강요 톤
- 영어 직역체 ("당신의 포트폴리오")

## 액션 버튼 톤

(stub)
- 등록: "관심종목" / "담기" / "등록"
- 해제: "해제" / "삭제"
- 새로고침: "새로고침" / "다시 시도"

## 알림 / 푸시 카피

(stub)
- 주체 명확히 (🇰🇷 한국장 / 🇺🇸 미국장)
- 다음 액션 한 줄 ("시나리오 한 번 점검!")
- 길이: 본문 ≤ 60자

## 면책

(stub) 실험지표 / AI 추천 등에 표시.
- 예: "매매 근거로 단독 사용 금지"
